
## Proposal Page Mobile Optimization

### Issues Identified

**1. ProposalDraft card header is cramped on mobile**
- The `CardHeader` uses `flex-row` with `items-start justify-between` always, pushing "Proposal Draft" title and BackupManager buttons side-by-side even on tiny screens
- BackupManager renders two buttons ("Save Backup" + "Backup Options") horizontally, which overflows on 390px screens

**2. BackupManager buttons overflow on mobile**
- Two buttons ("Save Backup" + "Backup Options") sit in a horizontal `flex` row with no wrapping
- On small screens these get squished or overflow

**3. CardContent padding too large on mobile**
- `p-6` on CardContent wastes 24px on each side (48px total) of a 390px screen

**4. Action buttons row is too tall on mobile**
- The 4 buttons (Add Section, Create Sections from Outline, Content Generation settings panel, Delete All Sections) stack via `flex-col sm:flex-row` which is fine, but the ContentGenerationButton renders an entire settings panel with switches, alerts, and coverage checks inline -- this pushes the section list far down the viewport

**5. SectionEditor wastes space on mobile**
- The section card header has title + delete button + expand chevron all on one line
- The title hover effect (`hover:bg-brand-green`) is desktop-oriented, not touch-friendly
- The "Generate with AI" button is right-aligned (`justify-end`) leaving dead space on mobile
- Textarea has `min-h-[200px]` which is fine but combined with padding eats most of the mobile viewport

**6. CompiledView has outer `p-6` padding**
- The CompiledView wrapper has `p-6` padding, on top of the Card's own padding, creating excessive spacing on mobile
- Card header buttons ("Show Raw" + "Copy") are small but squeezed on mobile

**7. AutoGeneratedProposal prerequisites grid uses `grid-cols-2`**
- The prerequisites checklist ("RFP Analysis Complete" + "Proposal Outline Ready") is a 2-column grid -- fine on desktop but text gets truncated on narrow mobile screens
- The "Generate Full Proposal" button sits in the header alongside the title, getting clipped on mobile
- The metadata stats grid uses `grid-cols-2 md:grid-cols-4` which is OK, but the cost optimization grid also uses `grid-cols-2 md:grid-cols-4` and can overflow
- Proposal content container has `p-6` padding that is excessive on mobile

**8. Tab triggers only show icons on mobile (no labels)**
- The UnifiedProposalView tabs hide labels on mobile (`hidden sm:inline`) leaving only icons, which is good, but the icons alone aren't very discoverable for first-time users

**9. SortableSection drag handle area**
- The drag handle (`pl-8`) wastes horizontal space on mobile where every pixel counts
- The grip icon is `opacity-0` until hover -- invisible on touch devices

### Implementation Plan

**File 1: `src/components/project/proposal-draft/ProposalDraft.tsx`**
- Change CardHeader to stack vertically on mobile: `flex-col sm:flex-row`
- Reduce CardContent padding on mobile: `p-3 sm:p-6`
- Title size: `text-xl sm:text-2xl`

**File 2: `src/components/project/proposal-draft/BackupManager.tsx`**
- Make button container wrap on mobile: `flex flex-wrap`
- Use icon-only buttons on mobile with tooltips, full text on sm+

**File 3: `src/components/project/proposal-draft/components/ContentGenerationButton.tsx`**
- Collapse the settings panel into a collapsible/disclosure on mobile -- show just the "Generate All Content" button by default
- Make the settings section collapsible with a "Settings" toggle
- Reduce padding in the settings box on mobile: `p-3 sm:p-4`

**File 4: `src/components/project/proposal-draft/SectionEditor.tsx`**
- Stack the "Generate with AI" button full-width on mobile instead of right-aligned
- Reduce textarea min-height on mobile: `min-h-[150px] sm:min-h-[200px]`
- Make save button full-width on mobile
- Improve touch target sizes for delete and expand buttons

**File 5: `src/components/project/proposal-draft/components/SortableSection.tsx`**
- Reduce drag handle padding on mobile: `pl-6 sm:pl-8`
- Show grip icon always on touch devices (opacity-100 on mobile, hover-controlled on desktop)

**File 6: `src/components/project/proposal-draft/components/CompiledView.tsx`**
- Remove outer `p-6` wrapper, use `p-2 sm:p-6`
- Stack header buttons vertically on mobile
- Reduce inner content padding: `p-3 sm:p-4`

**File 7: `src/components/project/auto-proposal/AutoGeneratedProposal.tsx`**
- Stack CardHeader title and button vertically on mobile
- Change prerequisites grid to `grid-cols-1 sm:grid-cols-2`
- Move "Generate Full Proposal" button below the header on mobile (full-width)
- Reduce content container padding: `p-3 sm:p-6`
- Stats grid: already `grid-cols-2 md:grid-cols-4`, keep as-is

**File 8: `src/components/project/unified-proposal/UnifiedProposalView.tsx`**
- Add short labels below icons on mobile tabs for discoverability (e.g., small text under each icon)
