
## Approve and Submit Proposal to Knowledge Base

### What This Feature Does

When you finish writing or generating a proposal, you can approve it and save it directly into your Knowledge Base. This means the app learns from every proposal you create -- your writing style, technical language, pricing approach, and past performance examples all get captured. Future proposals will draw on this growing library to produce better, more consistent content.

### How It Works

1. **Approve button appears** on both the Compiled Draft view and the Auto-Generated Proposal view once content exists
2. Clicking "Approve and Save to Knowledge Base" opens a confirmation dialog where you can:
   - Review the proposal title (auto-populated from the project)
   - Choose which Knowledge Base category to file it under (defaults to "Past Performance and Case Studies")
   - Optionally edit the title before saving
3. The proposal content is saved as a new Knowledge Base entry tied to your organization
4. A success message confirms the save, with a link to view the entry in your Knowledge Base

### Where the Button Appears

- **Compiled Draft tab**: Next to the existing "Copy" and "Show Raw/Formatted" buttons
- **Auto-Generated Proposal tab**: In the action buttons row alongside "Copy", "Export", and "Regenerate"

### Technical Details

**New Component: `SubmitToKnowledgeBaseButton.tsx`**
Location: `src/components/project/proposal-draft/components/`

- Accepts `proposalContent` (string), `projectId` (string), and optional `projectTitle` (string) props
- Renders a button with a `BookPlus` icon
- On click, opens an `AlertDialog` with:
  - A text input for the entry title (pre-filled with "[Project Title] - Approved Proposal")
  - A category selector dropdown populated from the existing `knowledgeCategories` array
  - "Cancel" and "Save to Knowledge Base" buttons
- On submit:
  - Fetches the current user session and their `current_organization_id` from the profiles table
  - Inserts a new row into `knowledge_entries` with:
    - `title`: user-chosen title
    - `content`: the full compiled/generated proposal markdown
    - `category`: selected category slug (e.g., `past-performance-&-case-studies`)
    - `user_id`: current user ID
    - `organization_id`: user's current org
    - `parsing_status`: `'completed'`
    - `parsing_progress`: `100`
  - Shows a success toast with a link to the Knowledge Base page

**Modified Files:**

1. **`CompiledView.tsx`**
   - Import and render `SubmitToKnowledgeBaseButton` in the header buttons row
   - Pass `compiledContent` and `projectId` as props

2. **`AutoGeneratedProposal.tsx`**
   - Import and render `SubmitToKnowledgeBaseButton` in the action buttons section (visible only when `generatedProposal` exists)
   - Pass `generatedProposal` and `projectId` as props

3. **`AutoGeneratedProposal.tsx` additional change**
   - Fetch project title from the existing project data to pass to the button for auto-populating the entry name

**No database changes required** -- the existing `knowledge_entries` table already has all the columns needed (title, content, category, user_id, organization_id, parsing_status, parsing_progress).

**No new edge functions required** -- the insert is a simple client-side Supabase SDK call, protected by existing RLS policies on `knowledge_entries`.
